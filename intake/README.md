# Intake

`intake/` generates the public daily intake page:

https://devlog.fivsevn.com/intake/

The page is a lightweight daily intake surface with three parts:

- a daily postcard
- a Daily Field Sample
- **On the shelf / 本日上架**, the source registry and RSS shelf

## Architecture

```text
intake/sources.md
    -> canonical hand-maintained source registry
    -> controls On the shelf / 本日上架

intake/build.py
    -> parses sources.md
    -> generates feeds.yml
    -> fetches RSS items
    -> assembles index.html

intake/scripts/daily_postcard.py
    -> generates the daily postcard fragment

intake/scripts/daily_field_sample.py
    -> generates the Daily Field Sample fragment

intake/feeds.yml
    -> generated intermediate file

intake/index.html
    -> generated public page
```

## Source registry

Edit:

```text
intake/sources.md
```

to change **On the shelf / 本日上架** sources, RSS URLs, and per-source fetch counts.

Each source uses this shape:

```yaml
name: Example Source
section: science_news
feed_url: https://example.com/feed.xml
source_cap: 6
site_url: https://example.com/
source_type: specialist_media
authority_level: specialist
reliability_score: 4
professional_value: 4
use_role: baseline
```

If a source has no usable feed, keep it as a website source:

```yaml
feed_url: ''
source_cap: 0
site_url: https://example.com/
```

## Spec reference

Field meanings and enum values are defined in `fivsevn-spec`:

```text
fivsevn-spec/source/spec-source-intake-fields-001.md
fivsevn-spec/appendix/spec-source-intake-enums-001.md
```

`intake/sources.md` is the current selected source list.

`fivsevn-spec` keeps the stable field rules and enum reference.

## Generated files

Do not hand-edit these as source files:

```text
intake/feeds.yml
intake/index.html
intake/data/daily_postcard.html
intake/data/daily_postcard.json
intake/data/daily_field_sample.html
intake/data/daily_field_sample.json
```

They are generated by scripts and may be overwritten by GitHub Actions.

## Manual build

From the repository root:

```bash
pip install -r intake/requirements.txt
python intake/scripts/daily_postcard.py
python intake/scripts/daily_field_sample.py
python intake/build.py
```

## GitHub Actions

The page is generated by:

```text
.github/workflows/build-news.yml
```

The workflow installs dependencies, runs the daily scripts, runs `build.py`, and commits generated outputs back to the repository.

## Editing boundary

```text
Change On the shelf sources, RSS URLs, or per-source fetch counts
-> edit intake/sources.md

Change page structure, HTML assembly, header/footer, or rendering logic
-> edit intake/build.py

Change the postcard module
-> edit intake/scripts/daily_postcard.py

Change the Daily Field Sample module
-> edit intake/scripts/daily_field_sample.py
```

`intake/` is a current-facing generated page, not a permanent archive or canonical content axis.
