# 预测市场

# 1. 基本模型

设一个预测市场网站为：

```text
X = {Q, C, M, A, P, R, G, S, L, Z}
```

其中：

| 字母 | 含义 | 要问的问题 |
|---|---|---|
| **X** | 网站 / 平台 | 这是哪个预测市场？ |
| **Q** | 预测问题集合 | 网站上有哪些预测问题？ |
| **C** | 创建机制 | 谁能创建问题？ |
| **M** | 参与方式 | 用户怎么参与？买卖、报概率、投票、讨论？ |
| **A** | 筹码 / 记账单位 | 用真钱、加密资产、虚拟币、积分，还是不用筹码？ |
| **P** | 价格 / 概率机制 | 概率或价格是怎么产生的？ |
| **R** | 结果确认机制 | 最后怎么判断事件是否发生？ |
| **G** | 争议处理机制 | 如果结果有争议，谁裁决？ |
| **S** | 奖惩机制 | 用户最后获得或失去什么？ |
| **L** | 法律 / 准入机制 | 是否 KYC？是否受监管？是否限制地区？ |
| **Z** | 题目范围 | 能预测什么类型的问题？ |

一个预测市场的完整结构可以表示为：

```text
X = {Q, C, M, A, P, R, G, S, L, Z}
```

不同预测市场网站之间的差异，可以表示为这些变量的不同取值。

---

# 2. 预测问题本身的结构

一个网站 X 里有很多预测问题：

```text
Q = {q₁, q₂, q₃, ...}
```

每个具体问题 q 又可以拆成：

```text
q = {E, T, B, D, H}
```

| 字母 | 含义 | 说明 |
|---|---|---|
| **E** | Event，事件 | 到底预测什么事情 |
| **T** | Time，时间 | 截止时间是什么 |
| **B** | Boundary，边界 | 什么算发生，什么不算发生 |
| **D** | Data source，数据来源 | 最后看哪个来源判定结果 |
| **H** | Hard cases，特殊情况 | 改名、延期、数据修正、模糊结果等怎么处理 |

例子：

> “公司 Y 是否会在 2026 年 12 月 31 日前上市？”

可以拆成：

```text
q = {
  E: 公司 Y 上市,
  T: 2026-12-31,
  B: 什么算上市,
  D: 交易所公告 / 公司公告 / 监管文件,
  H: SPAC、改名、延期、借壳等特殊情况怎么算
}
```

一个好的预测问题必须清楚定义 E、T、B、D、H。

坏问题：

> “某公司会不会成功？”

问题是：“成功”没有定义。

好问题：

> “某公司是否会在 2026 年全年实现营收超过 1 亿美元？”

这个问题有明确事件、时间、边界和数据来源，更适合做预测市场。

# 3. 变量字典

下面是模型中每个变量的常见取值。

---

## 3.1 C：谁创建问题

| 代码 | 含义 |
|---|---|
| **C₁** | 平台创建问题 |
| **C₂** | 用户自由创建问题 |
| **C₃** | 用户建议，平台审核后上线 |
| **C₄** | 研究团队 / 机构创建问题 |
| **C₅** | 去中心化自由创建问题 |

区别：

- **C₁**：像交易所上架产品。
- **C₂**：像社区发帖。
- **C₃**：像用户投稿，平台编辑审核。
- **C₄**：像研究项目或预测竞赛出题。
- **C₅**：像任何人都可以发行一个链上事件市场。

---

## 3.2 M：用户怎么参与

| 代码 | 含义 |
|---|---|
| **M₁** | 买卖 YES / NO 合约 |
| **M₂** | 买卖多结果合约 |
| **M₃** | 直接提交概率 |
| **M₄** | 投票 / 评论 / 讨论 |
| **M₅** | 提交问题、补充证据、参与治理 |

---

## 3.3 A：筹码 / 记账单位

| 代码 | 含义 |
|---|---|
| **A₀** | 无筹码 |
| **A₁** | 法币，例如美元 |
| **A₂** | 加密资产，例如 USDC |
| **A₃** | 虚拟币 / 积分 |
| **A₄** | 声誉分 / 预测分 |

区别：

- **A₁**：真钱，通常涉及合规和监管。
- **A₂**：链上资产，通常涉及钱包、智能合约、oracle。
- **A₃**：平台内积分，通常不能直接兑换现金。
- **A₄**：预测分数或声誉，用来衡量长期准确性。

---

## 3.4 P：价格 / 概率怎么产生

| 代码 | 含义 |
|---|---|
| **P₁** | 订单簿交易价格 |
| **P₂** | AMM / 自动做市价格 |
| **P₃** | 用户概率聚合 |
| **P₄** | 平台算法聚合 |
| **P₅** | 没有明确市场价格 |

解释：

- **P₁**：像股票交易，买卖双方挂单成交。
- **P₂**：由自动做市机制根据交易改变价格。
- **P₃**：很多人直接报概率，平台做平均或加权。
- **P₄**：平台用算法聚合不同用户的预测。
- **P₅**：没有正式价格，只是讨论或投票。

在买卖型预测市场（M₁）中，YES 价格通常可以粗略理解为市场隐含概率。

例如：

```text
YES = 0.63 美元
```

可以粗略理解为：

```text
市场认为事件发生概率约为 63%
```

但这不是纯概率，因为价格中还混有手续费、流动性、操纵风险、地区限制和交易者情绪。

---

## 3.5 R：结果怎么确认

| 代码 | 含义 |
|---|---|
| **R₁** | 平台管理员判定 |
| **R₂** | 指定官方数据源判定 |
| **R₃** | 市场创建者判定 |
| **R₄** | 预言机 / oracle 判定 |
| **R₅** | 研究团队 / 平台编辑判定 |

解释：

- **R₁**：平台最后拍板。
- **R₂**：看官方数据、交易所数据、公司公告、监管文件等。
- **R₃**：谁创建市场，谁解决市场。
- **R₄**：通过链上 oracle 把现实结果输入系统。
- **R₅**：研究平台或预测竞赛方按规则判定。

---

## 3.6 G：争议怎么处理

| 代码 | 含义 |
|---|---|
| **G₀** | 基本没有正式争议机制 |
| **G₁** | 平台最终裁决 |
| **G₂** | 社区 / moderator 介入 |
| **G₃** | oracle dispute 机制 |
| **G₄** | 仲裁机制 |
| **G₅** | 研究平台人工复核 |

预测市场最容易出问题的地方不是用户猜错，而是问题本身写得含糊。

比如：

> “某 AI 产品是否发布？”

争议可能是：

- 内测算不算发布？
- 只邀请用户算不算发布？
- 宣布但不能使用算不算发布？
- 改名发布算不算发布？

所以 **G** 颇为重要。

---

## 3.7 S：用户赢 / 输什么

| 代码 | 含义 |
|---|---|
| **S₁** | 现金收益 / 亏损 |
| **S₂** | 加密资产收益 / 亏损 |
| **S₃** | 虚拟币 / 积分 |
| **S₄** | 分数、排名、声誉 |
| **S₅** | 学习、讨论、预测记录 |

区别：

- 真钱市场：赢输的是钱。
- 虚拟币市场：赢输的是积分和排名。
- 预测训练平台：赢输的是分数、校准记录和声誉。
- 研究平台：重点是预测数据和长期准确性。

---

## 3.8 L：法律 / 准入机制

| 代码 | 含义 |
|---|---|
| **L₀** | 基本开放，不涉及真钱交易 |
| **L₁** | KYC / 地区限制 |
| **L₂** | 金融监管框架 |
| **L₃** | 加密 / 链上准入 |
| **L₄** | 学术 / 研究豁免或小额限制 |

这决定了一个预测市场到底更像：

- 游戏；
- 社区；
- 金融交易所；
- 博彩；
- 研究实验；
- 去中心化协议。

---

## 3.9 Z：题目范围

| 代码 | 含义 |
|---|---|
| **Z₁** | 政治 |
| **Z₂** | 金融 / 经济 |
| **Z₃** | 科技 / AI |
| **Z₄** | 体育 / 娱乐 |
| **Z₅** | 地缘政治 / 政策 |
| **Z₆** | 加密 / 链上 |
| **Z₇** | 任意社区问题 |
| **Z₈** | 学术 / 研究问题 |

- 只做政治预测，会像政治博彩 / 政治研究。
- 覆盖金融和经济，会像金融交易所。
- 开放任意问题，会像社区平台。
- 关注 AI、科技、地缘政治，会像预测训练平台。

---

# 4. 预测市场的统一流程

所有预测市场都可以压缩成这个流程：

```text
C → Q → M + A → P → R → G → S
```

意思是：

1. **C** 决定谁能创建问题；
2. 创建出一组 **Q**；
3. 用户通过 **M** 参与；
4. 使用 **A** 作为筹码或记录单位；
5. 系统生成 **P**，也就是价格或概率；
6. 到期后通过 **R** 确认结果；
7. 如果有争议，通过 **G** 处理；
8. 最后通过 **S** 给用户结算奖惩。

这就是预测市场的基本机器。

# 5. 用一个例子跑完整模型

假设网站 X 上有一个问题：

> “公司 Y 是否会在 2026 年 12 月 31 日前上市？”

问题 q 可以写成：

```text
q = {
  E: 公司 Y 上市,
  T: 2026-12-31,
  B: 在公开证券交易所挂牌交易才算上市，仅提交申请不算,
  D: 交易所公告、公司公告、监管文件,
  H: SPAC 算，改名后上市算，传闻不算
}
```

如果这是一个真钱交易市场：

```text
C = C₃
M = M₁
A = A₁
P = P₁
R = R₂
G = G₁
S = S₁
```

即：

- 用户建议问题，平台审核；
- 用户买 YES / NO；
- 用美元；
- 价格由订单簿产生；
- 最后看官方数据源；
- 有争议时平台裁决；
- 猜对赚钱，猜错亏钱。

假设 YES 价格是 0.30 美元。

你买 100 份 YES：

```text
成本 = 100 × 0.30 = 30 美元
```

如果公司 Y 上市：

```text
每份 YES 结算为 1 美元
收入 = 100 美元
利润 = 100 - 30 = 70 美元
```

如果公司 Y 没上市：

```text
每份 YES 结算为 0
损失 = 30 美元
```

如果是虚拟币社区市场，结构可能变成：

```text
C = C₂
M = M₁
A = A₃
P = P₁/P₂
R = R₃
G = G₂
S = S₃/S₄
```

如果是预测训练平台，结构可能变成：

```text
C = C₃/C₄
M = M₃
A = A₄
P = P₃/P₄
R = R₅
G = G₅
S = S₄/S₅
```

同一个事件，平台玩法不同，但基本流程相同。

# 6. 主要网站代入模型

下面用前面定义的字母表达主要预测市场网站。

---

## 6.1 [Polymarket](https://polymarket.com/)

```text
Polymarket = {Q, C₃, M₁/M₂, A₂, P₁/P₂, R₄, G₃, S₂, L₃, Z₁/Z₂/Z₃/Z₄/Z₆}
```

含义：

- 用户不能完全自由发题，偏 **C₃**；
- 用户用加密资产参与，属于 **A₂**；
- 主要买卖 YES / NO 或多结果合约，属于 **M₁/M₂**；
- 结果依赖 oracle，属于 **R₄/G₃**；
- 赢输的是链上资产，属于 **S₂**。

适合学习：

> 加密原生、开放式、互联网化的预测交易市场。

---

## 6.2 [Kalshi](https://kalshi.com/)

```text
Kalshi = {Q, C₃, M₁/M₂, A₁, P₁, R₁/R₂, G₁, S₁, L₁/L₂, Z₁/Z₂/Z₃/Z₄}
```

含义：

- 用户可以建议市场，但平台审核，属于 **C₃**；
- 用美元，属于 **A₁**；
- 买卖事件合约，属于 **M₁/M₂**；
- 价格由订单簿产生，属于 **P₁**；
- 受监管、准入更严格，属于 **L₁/L₂**。

适合学习：

> 合规金融化的 event contract 预测市场。

---

## 6.3 [PredictIt](https://www.predictit.org/)

```text
PredictIt = {Q, C₁/C₃, M₁, A₁, P₁, R₁/R₂, G₁, S₁, L₁/L₄, Z₁}
```

含义：

- 主要是政治预测，属于 **Z₁**；
- 用真钱，属于 **A₁**；
- 买卖政治事件份额，属于 **M₁**；
- 有教育 / 研究色彩和限制，属于 **L₄**。

适合学习：

> 老牌政治预测市场和教育研究型真钱预测市场。

---

## 6.4 [IBKR ForecastTrader / IBKR Prediction Markets](https://www.interactivebrokers.com/predictionmarkets/en/home.php)

```text
IBKR ForecastTrader = {Q, C₁/C₃, M₁/M₂, A₁, P₁, R₂, G₁, S₁, L₁/L₂, Z₁/Z₂/Z₄}
```

含义：

- 更像传统券商入口；
- 用户在经纪账户中接触事件合约；
- 用美元，属于 **A₁**；
- 受金融监管框架影响，属于 **L₂**。

适合学习：

> 传统券商如何把预测市场包装成可交易金融产品入口。

---

## 6.5 [Manifold](https://manifold.markets/)

```text
Manifold = {Q, C₂, M₁/M₄/M₅, A₃, P₁/P₂, R₃, G₂, S₃/S₄, L₀, Z₇}
```

含义：

- 用户自由创建市场，属于 **C₂**；
- 用户用虚拟币 / mana 参与，属于 **A₃**；
- 可以买卖预测份额，也可以讨论、补充信息，属于 **M₁/M₄/M₅**；
- 通常由市场创建者结算，属于 **R₃**；
- 有争议时社区或 moderator 介入，属于 **G₂**；
- 题目范围很开放，属于 **Z₇**。

适合学习：

> 社区型、游戏化、无真钱预测市场。

---

## 6.6 [Metaculus](https://www.metaculus.com/)

```text
Metaculus = {Q, C₃/C₄, M₃/M₅, A₄, P₃/P₄, R₅, G₅, S₄/S₅, L₀, Z₃/Z₅/Z₈}
```

含义：

- 很多问题由平台、编辑或研究相关方创建，也可由用户建议，属于 **C₃/C₄**；
- 用户直接提交概率，属于 **M₃**；
- 核心不是筹码，而是预测分和声誉，属于 **A₄**；
- 概率由用户预测和平台算法聚合，属于 **P₃/P₄**；
- 适合科技、AI、政策、长期预测等问题，属于 **Z₃/Z₅/Z₈**。

适合学习：

> 概率预测、校准、Brier score、长期预测能力训练。

---

## 6.7 [Good Judgment Open](https://www.gjopen.com/)

```text
Good Judgment Open = {Q, C₄, M₃, A₄, P₃/P₄, R₅, G₅, S₄/S₅, L₀, Z₅/Z₈}
```

含义：

- 问题偏机构化和研究化，属于 **C₄**；
- 用户直接提交概率，属于 **M₃**；
- 使用预测分、排名和声誉，属于 **A₄/S₄**；
- 重点是 superforecasting 训练。

适合学习：

> 超级预测者方法、地缘政治和政策预测训练。

---

## 6.8 [Iowa Electronic Markets](https://iem.uiowa.edu/iem/), [IEM](https://iem.uiowa.edu/iem/)

```text
IEM = {Q, C₄, M₁/M₂, A₁, P₁, R₂/R₅, G₅, S₁, L₄, Z₁/Z₂/Z₈}
```

含义：

- 学术项目方创建市场，属于 **C₄**；
- 小额真钱交易，属于 **A₁ + L₄**；
- 以政治和经济类问题为主，属于 **Z₁/Z₂/Z₈**。

适合学习：

> 预测市场历史、学术实验、小额真钱市场。

---

## 6.9 [Augur](https://www.augur.net/)

```text
Augur = {Q, C₅, M₁/M₂, A₂, P₁/P₂, R₄, G₃, S₂, L₃, Z₇}
```

含义：

- 去中心化自由创建市场，属于 **C₅**；
- 使用链上资产，属于 **A₂**；
- 通过 oracle 确认结果，属于 **R₄**；
- 有 dispute 机制，属于 **G₃**。

适合学习：

> 去中心化 oracle、争议处理和无需中心化平台的预测市场。

---

## 6.10 [Omen](https://omen.eth.link/) / [Gnosis](https://www.gnosis.io/) 系

```text
Omen/Gnosis = {Q, C₅, M₁/M₂, A₂, P₂, R₄, G₃/G₄, S₂, L₃, Z₇}
```

含义：

- 更像协议和基础设施；
- 用户可以基于条件代币框架创建市场；
- 使用链上资产；
- 重点是 AMM、conditional tokens、oracle 和仲裁。

适合学习：

> 去中心化预测市场基础设施和链上市场设计。


# 10. 最后总结

最核心的一句话：

```text
X = {Q, C, M, A, P, R, G, S, L, Z}
```

看任何预测市场网站，都问这十个问题：

1. **Q**：它有什么预测问题？
2. **C**：谁能创建问题？
3. **M**：用户怎么参与？
4. **A**：用什么筹码或记账单位？
5. **P**：价格或概率怎么产生？
6. **R**：结果怎么确认？
7. **G**：争议怎么处理？
8. **S**：用户赢输什么？
9. **L**：法律和准入机制是什么？
10. **Z**：题目范围是什么？

---

note：预测市场是一个外壳。你不一定要下注赌钱、或者过度地参与到他人事情的纠葛中去。了解预测市场的价值在于培养理性的思维方式。普通讨论里大家说“很可能”“不太可能”“一定会”“大概率”，这些话很模糊、难追责，消磨时间精力又无法训练思维能力；如果是预测市场的话，会把一个问题抽出变成“某件事被认为发生的概率在57%”，这个数字可以被记录、被比较、随着信息的更新而进一步校准、聚合起来进行综合判断、事后验证、辅助决策，等等。普通讨论常常会夹带太多的立场、情绪、论证和胜负；但预测市场只有：问题定义 → 概率判断 → 信息更新 → 结果验证 → 复盘校准。你要训练的不是下注能力，而是不把愿望当预测、不把立场当事实、承认未来有概率分布、愿意根据证据修改判断，等等能力。
